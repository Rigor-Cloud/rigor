  <action>
Replace the `axum::serve(listener, app)` usage in both `TestProxy::start()` and `TestProxy::start_with_mock()` with a manual accept loop using `hyper_util::server::conn::auto::Builder::new(hyper_util::rt::TokioExecutor::new()).serve_connection_with_upgrades(io, service)`. This matches the pattern from production `start_daemon()` (daemon/mod.rs lines 542-545).

The change:
1. Instead of `axum::serve(listener, app).with_graceful_shutdown(...)`, spawn a task that:
   - Runs an accept loop on the TcpListener
   - For each accepted connection, wraps in `hyper_util::rt::TokioIo`
   - Creates a `hyper::service::service_fn` that converts the request to axum format (same pattern as daemon/mod.rs lines 523-536) and calls the router
   - Calls `serve_connection_with_upgrades(io, service).await`
   - Respects the shutdown_rx oneshot for graceful shutdown (use `tokio::select!` on accept + shutdown signal)

2. Add necessary imports: `hyper`, `hyper_util`, `tower::Service`

3. Keep the existing `ENV_MUTEX`, `TestProxy` struct fields, and `Drop` implementation unchanged.

4. Existing unit tests (`test_proxy_starts_on_ephemeral_port`, `test_proxy_url_format`) must still pass -- they don't use CONNECT, just basic connectivity.

IMPORTANT: Do NOT modify the `TestProxy` struct fields or the public API. Only change the internal server loop from `axum::serve` to the hyper_util builder pattern.

Reference the production pattern in daemon/mod.rs:
```rust
let service = hyper::service::service_fn(
    move |req: hyper::Request<hyper::body::Incoming>| {
        let app = app.clone();
        async move {
            use tower::Service;
            let (parts, incoming) = req.into_parts();
            let body = axum::body::Body::new(incoming);
            let req = axum::http::Request::from_parts(parts, body);
            let mut svc = app;
            let resp = svc.call(req).await.unwrap();
            Ok::<_, std::convert::Infallible>(resp)
        }
    },
);
hyper_util::server::conn::auto::Builder::new(
    hyper_util::rt::TokioExecutor::new(),
)
.serve_connection_with_upgrades(io, service)
.await
```

For graceful shutdown with the accept loop, use `tokio::select!`:
```rust
loop {
    tokio::select! {
        result = listener.accept() => {
            let (stream, _addr) = result.unwrap();
            let app = app.clone();
            tokio::spawn(async move {
                let io = hyper_util::rt::TokioIo::new(stream);
                let service = /* service_fn wrapping app */;
                let _ = hyper_util::server::conn::auto::Builder::new(
                    hyper_util::rt::TokioExecutor::new(),
                )
                .serve_connection_with_upgrades(io, service)
                .await;
            });
        }
        _ = &mut shutdown_rx => { break; }
    }
}
```

**Test 1: `blind_tunnel_non_llm_host`**
Exercises the blind-tunnel path (proxy.rs lines 639-688) where `should_mitm_target()` returns false.

Steps:
1. Start a TCP echo server on an ephemeral port that reads data and echoes it back.
2. Start a `TestProxy::start(MINIMAL_YAML)` (no MITM needed, MITM_ENABLED defaults to false).
3. Open a raw TCP connection to the TestProxy address.
4. Send a manual CONNECT request: `CONNECT 127.0.0.1:{echo_port} HTTP/1.1\r\nHost: 127.0.0.1:{echo_port}\r\n\r\n`
5. Read the response and assert it contains `200` (tunnel established).
6. Send arbitrary bytes through the tunnel (e.g., `b"hello blind tunnel"`).
7. Read the echoed bytes and assert they match exactly.

This proves the proxy's blind-tunnel `copy_bidirectional` path works end-to-end.

**Test 2: `mitm_tls_handshake_validates_against_ca`**
Exercises the MITM TLS path (proxy.rs lines 690-823) where `should_mitm_target()` returns true.

Steps:
1. Start a `MockLlmServerBuilder::new().anthropic_chunks("test response").build().await` on an ephemeral port.
2. Start a `TestProxy::start_with_mock(MINIMAL_YAML, &mock.url())`.
3. Read the CA cert PEM from `proxy.home.rigor_dir.join("ca.pem")` -- this was written by `RigorCA::load_or_generate()` during TestProxy startup.
4. Build a `rustls::ClientConfig` that trusts only this CA cert (same pattern as `TestCA::client_config()` -- parse PEM, add to RootCertStore, build ClientConfig).
5. Save and set `MITM_ENABLED` to true via `rigor::daemon::ws::set_mitm_enabled(true)` (use a guard to restore on drop). Use a `static Mutex` for serialization like daemon/mod.rs tests do.
6. Open a raw TCP connection to the TestProxy address.
7. Send CONNECT: `CONNECT api.anthropic.com:443 HTTP/1.1\r\nHost: api.anthropic.com:443\r\n\r\n`
8. Read the 200 response.
9. Perform a TLS handshake on the upgraded stream using `tokio_rustls::TlsConnector` with the client config from step 4 and `ServerName::try_from("api.anthropic.com")`.
10. Assert the TLS handshake succeeds (no error) -- this proves the proxy generated a valid cert chain for api.anthropic.com signed by the test CA.
11. Optionally: send an HTTP request over the TLS stream to verify the MockLlmServer responds (proves the full MITM pipeline works).
12. Restore MITM_ENABLED to original value.

**Shared helpers in the test file:**
- `const MINIMAL_YAML` -- same as used in harness_smoke.rs
- `async fn start_echo_server() -> (SocketAddr, JoinHandle)` -- spawns a TCP echo server
- `async fn send_connect(proxy_addr: SocketAddr, target: &str) -> TcpStream` -- sends CONNECT, reads 200, returns upgraded stream
- `fn load_ca_cert_from_file(ca_pem_path: &Path) -> rustls::ClientConfig` -- reads PEM, builds client config

**Important notes:**
- Use `tokio::io::{AsyncReadExt, AsyncWriteExt}` for raw TCP I/O.
- Use `tokio::net::TcpStream` for the connection to proxy.
- For the MITM test, the echo server is NOT needed -- the MockLlmServer receives the proxied request.
- The CONNECT to `api.anthropic.com:443` doesn't go to the real API -- the proxy will try to `TcpStream::connect("api.anthropic.com:443")` for the upstream, but since we're using `start_with_mock`, the proxy routes to the mock via `RIGOR_TARGET_API`. HOWEVER: the CONNECT blind/MITM decision happens based on the target hostname, but the actual upstream connection in the MITM path goes through the proxy's axum router which uses `RIGOR_TARGET_API`. For the blind tunnel path, the proxy does `TcpStream::connect(&target)` directly -- so for the echo test, use 127.0.0.1:PORT as the target.
- For the MITM test, the proxy terminates TLS and serves its axum router on the decrypted stream. The axum router then proxies to `RIGOR_TARGET_API` (the mock). So after the TLS handshake, you can send an HTTP POST to `/v1/messages` and get the mock response back.
- Use `#[tokio::test]` for both tests.
- Add `use std::sync::Mutex;` and `static MITM_LOCK: Mutex<()> = Mutex::new(());` to serialize MITM tests (matching the pattern in daemon/mod.rs tests).

**Read timeout:** Add a 5-second timeout on reads from the tunnel to prevent test hangs. Use `tokio::time::timeout`.
