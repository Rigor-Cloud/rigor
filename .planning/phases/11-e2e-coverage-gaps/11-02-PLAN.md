Hook evaluation pipeline (lib.rs run_hook()):
1. Reads StopHookInput from stdin (JSON)
2. Checks daemon_alive() -- if no daemon, writes allow response
3. Loads rigor.yaml from cwd
4. Builds ArgumentationGraph, computes DF-QuAD strengths
5. Builds EvaluatorPipeline with RegexEvaluator
6. Extracts claims (or uses RIGOR_TEST_CLAIMS env var override)
7. Evaluates claims against constraints
8. Determines decision (block/warn/allow) based on violation severity

RIGOR_TEST_CLAIMS override (from lib.rs):
- When set, claims are parsed from this env var instead of transcript extraction
- Format: JSON array of Claim objects with id, text, confidence, claim_type fields

  <action>
Create `crates/rigor/tests/pid_lifecycle_e2e.rs` with E2E tests exercising the PID file lifecycle through `RIGOR_HOME` isolation. These tests exercise the scenario where a daemon writes a PID, "crashes" (PID file left behind), and a new daemon start correctly detects the stale PID and overwrites it.

IMPORTANT: These tests mutate the global RIGOR_HOME env var, so they MUST use `crate paths::RIGOR_HOME_TEST_LOCK` (or replicate the pattern) for serialization. Since this is an integration test file (not inside the rigor crate's `mod tests`), we cannot access `crate::paths::RIGOR_HOME_TEST_LOCK`. Instead, define a local static mutex and use `std::env::set_var` / `std::env::remove_var` with save/restore, just like the daemon/mod.rs tests do.

**Test 1: `pid_file_crash_recovery_lifecycle`**
Exercises the full write-crash-rewrite lifecycle:
1. Create a TempDir as RIGOR_HOME.
2. Save original RIGOR_HOME, set to temp path (under mutex guard).
3. Call `write_pid_file()` -- writes current process PID.
4. Assert `daemon_alive()` returns true.
5. Simulate crash: manually overwrite the PID file with a dead PID (2000000).
6. Assert `daemon_alive()` returns false (stale PID detected).
7. Call `write_pid_file()` again -- overwrites the stale PID with current process PID.
8. Assert `daemon_alive()` returns true again.
9. Call `remove_pid_file()`.
10. Assert `daemon_alive()` returns false.
11. Restore original RIGOR_HOME.

This proves the full lifecycle: start -> crash -> detect stale -> restart -> clean shutdown.

**Test 2: `pid_file_absent_directory_created`**
Exercises the case where the .rigor directory doesn't exist yet:
1. Create a TempDir but do NOT create .rigor subdir.
2. Set RIGOR_HOME to that TempDir (it points at the .rigor dir which doesn't exist).
3. Call `write_pid_file()`.
4. Assert it succeeds (write_pid_file creates parent directories via create_dir_all).
5. Assert the PID file exists.
6. Assert `daemon_alive()` returns true.
7. Cleanup.

**Test 3: `pid_file_overwrite_is_atomic`**
1. Create TempDir, set RIGOR_HOME.
2. Write a PID file with PID 2000000 (dead).
3. Assert `daemon_alive()` returns false.
4. Call `write_pid_file()` (overwrites with current PID).
5. Read the file content. Assert it contains exactly the current process ID.
6. Assert `daemon_alive()` returns true.
7. Cleanup.

**Shared helper:**
```rust
use std::sync::Mutex;
use rigor::daemon::{write_pid_file, remove_pid_file, daemon_alive, daemon_pid_file};

static PID_TEST_LOCK: Mutex<()> = Mutex::new(());

fn with_temp_rigor_home<F: FnOnce(&std::path::Path)>(f: F) {
    let _guard = PID_TEST_LOCK.lock().unwrap_or_else(|e| e.into_inner());
    let original = std::env::var("RIGOR_HOME").ok();
    let tmp = tempfile::TempDir::new().unwrap();
    let rigor_dir = tmp.path().to_path_buf();
    std::fs::create_dir_all(&rigor_dir).unwrap();
    unsafe { std::env::set_var("RIGOR_HOME", &rigor_dir) };

    f(&rigor_dir);

    match original {
        Some(v) => unsafe { std::env::set_var("RIGOR_HOME", v) },
        None => unsafe { std::env::remove_var("RIGOR_HOME") },
    }
}
```

IMPORTANT: `RIGOR_HOME` in daemon code points to the `.rigor` directory itself (not the HOME parent). So set RIGOR_HOME to the TempDir path directly (not to a `.rigor` subdir). Check what `daemon_pid_file()` returns: `Some(rigor_home().join("daemon.pid"))` where `rigor_home()` reads `RIGOR_HOME` env var. So RIGOR_HOME should be the directory where `daemon.pid` will be written.

All tests use `#[test]` (synchronous). Run with `--test-threads=1` if needed due to global env var mutation, but the mutex should serialize them.
