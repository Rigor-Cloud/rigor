**3. Implement crates/rigor-harness/src/proxy.rs -- TestProxy:**

```rust
use crate::home::IsolatedHome;
use std::net::SocketAddr;
use std::sync::{Arc, Mutex};
use tokio::sync::oneshot;

pub struct TestProxy {
    pub addr: SocketAddr,
    pub home: IsolatedHome,
    shutdown_tx: Option<oneshot::Sender<()>>,
    _handle: tokio::task::JoinHandle<()>,
}
```

`TestProxy::start(rigor_yaml: &str) -> Self`:
1. Create `IsolatedHome::new()`
2. Write rigor.yaml via `home.write_rigor_yaml(rigor_yaml)`
3. **Critical HOME isolation:** Since `DaemonState::load()` calls `RigorCA::load_or_generate()` which uses `dirs::home_dir()` / `std::env::var("HOME")`, and since we cannot safely call `std::env::set_var` in parallel tests, TestProxy uses **a dedicated thread** that:
   a. Sets HOME in its own scope via `unsafe { std::env::set_var("HOME", home.home_str()) }` (edition 2021 -- technically safe, logically unsound but acceptable in a single-threaded spawned context)
   b. Calls `DaemonState::load(yaml_path, event_tx)`
   c. Sends the state back via a channel
   
   **Alternative (preferred if feasible):** Use `DaemonState::empty(event_tx)` to avoid the HOME-sensitive code path, then bind the router. This works because the smoke test just needs a running proxy endpoint, not the full RigorCA. If `DaemonState::load()` is needed (because it parses rigor.yaml constraints), use `std::env::set_var` in a `std::thread::spawn` scope and communicate back via `std::sync::mpsc::channel`.

   Actually, the cleanest approach: set the `RIGOR_TARGET_API` env var on the _test_ level (callers of TestProxy do this), and for HOME isolation, since `DaemonState::load` reads HOME for RigorCA::load_or_generate, the TestProxy must ensure HOME is set to the IsolatedHome path. Since TestProxy tests should use `#[tokio::test(flavor = "multi_thread")]` and there's inherent risk with env var mutation, the pragmatic solution is:
   - TestProxy::start is `pub async fn start(rigor_yaml: &str) -> Self`
   - Inside, use a `tokio::task::spawn_blocking` closure that:
     1. Saves original HOME
     2. Sets HOME to isolated path via `unsafe { std::env::set_var(...) }`
     3. Creates DaemonState::load
     4. Restores original HOME
     5. Returns the state
   - Then bind the router to ephemeral port in an async task

4. Create `(event_tx, _) = rigor::daemon::ws::create_event_channel()`
5. Load state, wrap in `Arc<Mutex<DaemonState>>` (= SharedState)
6. Call `rigor::daemon::build_router(shared)`
7. Bind `tokio::net::TcpListener::bind("127.0.0.1:0").await`
8. Extract addr
9. Create oneshot shutdown channel
10. `tokio::spawn` the axum::serve with `with_graceful_shutdown`
11. Return `TestProxy { addr, home, shutdown_tx, _handle }`

Also implement:
- `pub fn url(&self) -> String` -> `format!("http://{}", self.addr)`
- `pub fn addr(&self) -> SocketAddr` -> `self.addr`
- `impl Drop for TestProxy` -> send shutdown via `self.shutdown_tx.take().map(|tx| let _ = tx.send(()))`

Also implement `TestProxy::start_with_mock(rigor_yaml: &str, mock_url: &str) -> Self` which sets `RIGOR_TARGET_API` to the mock URL before creating DaemonState. This is the common usage pattern -- point the proxy at a MockLlmServer.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TestProxy env::set_var -> parallel tests | HOME mutation in spawn_blocking is time-limited but still a race window |
| Test subprocess -> real binary | CARGO_BIN_EXE_rigor runs the actual binary, which could touch real HOME if not isolated |
| Mock upstream -> proxy forwarding | Proxy forwards to mock; if RIGOR_TARGET_API not set, proxy would hit real Anthropic API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Tampering | TestProxy HOME env var race in parallel tests | mitigate | Use spawn_blocking with save/restore pattern; document that TestProxy tests should not run in parallel with other HOME-sensitive tests; integration test binary (harness_smoke) runs in its own process |
| T-07-07 | Information Disclosure | Subprocess run_rigor might leak real API keys to mock server | mitigate | subprocess helpers env_remove RIGOR_TEST_CLAIMS; TestProxy::start_with_mock sets RIGOR_TARGET_API to mock URL ensuring no real API calls |
| T-07-08 | Tampering | DaemonState::load writes CA to IsolatedHome instead of real HOME | mitigate | TestProxy sets HOME to IsolatedHome.path before DaemonState::load; ca.pem written to TempDir which is cleaned on Drop |
