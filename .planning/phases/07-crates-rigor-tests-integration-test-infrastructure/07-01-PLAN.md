**home.rs -- IsolatedHome:**
```rust
use std::path::PathBuf;

pub struct IsolatedHome {
    _temp: tempfile::TempDir,
    pub path: PathBuf,
    pub rigor_dir: PathBuf,
}

impl IsolatedHome {
    pub fn new() -> Self {
        let temp = tempfile::TempDir::new().expect("failed to create temp HOME");
        let path = temp.path().to_path_buf();
        let rigor_dir = path.join(".rigor");
        std::fs::create_dir_all(&rigor_dir).expect("failed to create .rigor dir");
        Self { _temp: temp, path, rigor_dir }
    }

    /// Write a rigor.yaml into the isolated home directory
    pub fn write_rigor_yaml(&self, content: &str) -> PathBuf {
        let yaml_path = self.path.join("rigor.yaml");
        std::fs::write(&yaml_path, content).expect("write rigor.yaml");
        yaml_path
    }

    /// Get HOME value suitable for Command::env("HOME", ...)
    pub fn home_str(&self) -> String {
        self.path.to_string_lossy().to_string()
    }

    /// Get the .rigor directory path (for CA certs, PID files, etc.)
    pub fn rigor_dir_str(&self) -> String {
        self.rigor_dir.to_string_lossy().to_string()
    }
}

impl Default for IsolatedHome {
    fn default() -> Self { Self::new() }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test CA -> real trust store | Test CA must never leak outside TempDir |
| Ephemeral port -> localhost | Mock servers bind only to 127.0.0.1, not 0.0.0.0 |
| Test HOME -> real HOME | IsolatedHome must never mutate real ~/.rigor/ |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Elevation of Privilege | TestCA cert leaking to system trust store | mitigate | TestCA is ephemeral in-memory only; IsolatedHome TempDir cleanup on Drop ensures ca.pem never persists outside test scope |
| T-07-02 | Tampering | Tests modifying real ~/.rigor/ (PID file, CA cert, violations log) | mitigate | IsolatedHome creates TempDir-based HOME; subprocess tests use Command::env("HOME", ...) per invariants.rs B10 pattern; IsolatedHome does NOT call std::env::set_var globally |
| T-07-03 | Spoofing | Ephemeral port hijacked between bind and connect | accept | Bind to 127.0.0.1 only (loopback); standard OS-assigned port pattern; risk is vanishingly low on localhost |
| T-07-04 | Tampering | Parallel tests racing on HOME env var | mitigate | IsolatedHome does NOT call std::env::set_var; uses Command::env for subprocess isolation only; in-process tests pass paths explicitly |
| T-07-05 | Denial of Service | Mock server leaked task hangs test suite | mitigate | oneshot shutdown channel + Drop implementation sends shutdown signal; with_graceful_shutdown on axum::serve |
</threat_model>
