  <action>
Add a new step to the existing `clippy` job in `.github/workflows/ci.yml`. The step should come AFTER the "Run Clippy" step and BEFORE any other jobs. It does not need its own job -- it is a zero-cost grep that runs in milliseconds.

Add this step:

```yaml
      - name: Guard against raw HOME for .rigor paths
        run: |
          # All .rigor/ path resolution must go through crate::paths::rigor_home().
          # Category B sites (non-.rigor HOME usage) are allowlisted with '// rigor-home-ok'.
          VIOLATIONS=$(grep -rn 'dirs::home_dir\|env::var("HOME")\|env::var_os("HOME")' \
            crates/rigor/src/ \
            --include='*.rs' \
            | grep -v 'src/paths.rs' \
            | grep -v 'rigor-home-ok' \
            || true)
          if [ -n "$VIOLATIONS" ]; then
            echo "::error::Raw HOME access found outside paths.rs. Use crate::paths::rigor_home() instead, or add '// rigor-home-ok' for non-.rigor/ paths."
            echo ""
            echo "$VIOLATIONS"
            exit 1
          fi
```

Place it as the last step in the clippy job, after "Run Clippy". The step uses `::error::` GitHub Actions annotation for clear CI output.

Do NOT modify any other jobs or steps in ci.yml.
