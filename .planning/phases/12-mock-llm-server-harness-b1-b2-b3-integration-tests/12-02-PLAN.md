Create `crates/rigor/tests/b1_kill_switch.rs` -- an integration test proving the
proxy BLOCK path drops upstream and injects an error SSE event when RIGOR_NO_RETRY=1.

File structure:

```rust
//! B1: Streaming kill-switch integration test.
//!
//! Proves: when the proxy detects a constraint violation mid-stream and
//! RIGOR_NO_RETRY=1, it drops the upstream connection and injects an
//! `event: error` SSE event into the client stream containing "rigor BLOCKED".
```

1. **BLOCK_CONSTRAINT_YAML constant:**
   Use the same Rego keyword constraint pattern from stop_hook_e2e.rs (above in interfaces).
   The constraint name/description must contain words > 3 chars that appear in the
   mock SSE text. The word "violation_marker" (lowercased) extracted from the description
   "Blocks if claim text contains VIOLATION_MARKER" will be a keyword. The SSE text
   must contain "violation_marker" (case-insensitive match) at a sentence boundary.

2. **VIOLATION_TEXT constant:**
   The text MockLlmServer will serve as SSE. Must satisfy three pre-filter conditions:
   (a) contain a keyword from constraint description (e.g. "VIOLATION_MARKER"),
   (b) have a sentence boundary (". "),
   (c) be longer than 20 chars.
   Use: `"The system contains VIOLATION_MARKER in its output. This is a factual statement."`
   This text is declarative (not a question/hypothetical) so the heuristic extractor
   treats it as an assertion.

3. **Helper functions:**
   Copy the `anthropic_request_body()` and `proxy_post()` helpers from proxy_hotpath.rs.

4. **ENV_MUTEX for RIGOR_NO_RETRY:**
   Because `RIGOR_NO_RETRY` is a process-global env var checked at BLOCK time (not
   startup), it must be set before the request and restored after. Use a static Mutex:
   ```rust
   static ENV_LOCK: std::sync::Mutex<()> = std::sync::Mutex::new(());
   ```
   Inside each test: acquire lock, save original value, set RIGOR_NO_RETRY=1, run test,
   restore, drop lock. This prevents races with B2 tests which must NOT have this set.

5. **Test: `b1_block_drops_upstream_and_injects_error_sse`**
   ```
   #[tokio::test]
   async fn b1_block_drops_upstream_and_injects_error_sse() {
       let _guard = ENV_LOCK.lock().unwrap();
       let orig = std::env::var("RIGOR_NO_RETRY").ok();
       unsafe { std::env::set_var("RIGOR_NO_RETRY", "1") };

       let mock = MockLlmServerBuilder::new()
           .anthropic_chunks(VIOLATION_TEXT)
           .build()
           .await;
       let proxy = TestProxy::start_with_mock(BLOCK_CONSTRAINT_YAML, &mock.url()).await;
       let body = anthropic_request_body(true, "Tell me something");
       let resp = proxy_post(&proxy.url(), &body).await;
       let resp_body = resp.text().await.unwrap();

       // Restore env
       match orig {
           Some(v) => unsafe { std::env::set_var("RIGOR_NO_RETRY", v) },
           None => unsafe { std::env::remove_var("RIGOR_NO_RETRY") },
       }

       // BLOCK should inject error SSE event
       assert!(
           resp_body.contains("event: error"),
           "Should contain error SSE event. Got:\n{}",
           resp_body
       );
       assert!(
           resp_body.contains("rigor BLOCKED"),
           "Error event should mention 'rigor BLOCKED'. Got:\n{}",
           resp_body
       );
   }
   ```

6. **Test: `b1_block_returns_200_status`**
   A second test verifying HTTP 200 status code (SSE stream already started before
   BLOCK, so the HTTP status is 200, and the error is communicated via SSE event).
   Same setup as above, but assert `resp.status() == 200` separately.

CRITICAL CONSTRAINTS:
- The constraint `id` in the Rego MUST match the constraint_id in the violation output.
- The SSE text MUST be an assertion (not question, not hypothetical) -- see heuristic
  extractor filtering in the interfaces section.
- Use `unsafe` for `std::env::set_var`/`remove_var` (required in Rust 2024 edition).

5. **Request body with system prompt:**
   The retry path appends feedback to `body["system"]`. For the feedback to be
   appended (not silently skipped), the original request MUST contain a `"system"` field:
   ```rust
   let body = serde_json::json!({
       "model": "claude-sonnet-4-20250514",
       "max_tokens": 100,
       "stream": true,
       "system": "You are a helpful assistant.",
   ```

   - Assertions on the response body:

       Use `extract_text_from_sse` to verify the extracted text contains words
       from CLEAN_TEXT.
     - Response should NOT contain "event: error" (retry succeeded).
   - Assertions on mock.received_requests():
     - `received.len() >= 2` (original request + retry).
     - `received[1].body["system"]` contains `"[RIGOR EPISTEMIC CORRECTION]"`

     - `received[1].body["system"]` contains `"TRUTH:"` -- verifying truth

     - `received.len() == 2` (original + exactly one retry, not more).
