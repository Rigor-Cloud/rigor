<objective>
Enhance MockLlmServer with request tracking and response-sequence capabilities
so B1/B2/B3 integration tests (Plan 02) can inspect forwarded request bodies
and receive different SSE responses per call.

Purpose: REQ-022 requires the mock-LLM server harness to serve deterministic SSE
responses configurable per-test. The existing MockLlmServer serves a single fixed
response. B2 needs call 0 = violation, call 1 = clean. B3 needs to inspect what
the proxy forwarded upstream.

Output: Enhanced MockLlmServer with `received_requests()` accessor and
`response_sequence()` builder method.
</objective>

Add request tracking to MockLlmServer. This is an enhancement, not a rebuild --
the existing builder API (anthropic_chunks, openai_chunks, raw_chunks, route) and
MockLlmServer API (start, addr, url) remain unchanged.

1. Add a `ReceivedRequest` struct at the top of the file:
```rust
#[derive(Debug, Clone)]
pub struct ReceivedRequest {
    pub body: serde_json::Value,
}
```

2. Add a `received` field to `MockLlmServer`:
```rust
pub struct MockLlmServer {
    addr: SocketAddr,
    shutdown_tx: Option<oneshot::Sender<()>>,
    _handle: tokio::task::JoinHandle<()>,
    received: Arc<Mutex<Vec<ReceivedRequest>>>,
}
```
Use `std::sync::{Arc, Mutex}` (already imported: Arc is imported, add Mutex).

3. Add a `received_requests()` accessor:
```rust
impl MockLlmServer {
    pub fn received_requests(&self) -> Vec<ReceivedRequest> {
        self.received.lock().unwrap().clone()
    }
}
```

4. Add `response_sequence()` to `MockLlmServerBuilder`:
```rust
pub struct MockLlmServerBuilder {
    chunks: Vec<String>,
    response_sequence: Option<Vec<Vec<String>>>,
    route_path: String,
}

impl MockLlmServerBuilder {
    pub fn response_sequence(mut self, responses: Vec<Vec<String>>) -> Self {
        self.response_sequence = Some(responses);
        self
    }
}
```
Initialize `response_sequence: None` in `new()`.

5. Update `build()` to:
   - Create `Arc<Mutex<Vec<ReceivedRequest>>>` for tracking.
   - Use `axum::extract::Bytes` (not `axum::body::Bytes`) to receive the request body
     in the handler. The handler signature becomes:
     `move |body: axum::body::Bytes| { ... }`
   - Parse the body as JSON and push `ReceivedRequest { body }` into the tracking vec.
     If JSON parse fails, push with `serde_json::Value::Null`.
   - Use `AtomicUsize` call counter for response selection when `response_sequence`
     is `Some`. Call index selects the response set; if index >= len, use last entry.
     When `response_sequence` is `None`, use `self.chunks` (backward compatible).
   - Pass `received` into `MockLlmServer`.

6. Update `MockLlmServer::start()` to pass `received: Arc::new(Mutex::new(Vec::new()))`.
   Since `start()` calls `MockLlmServerBuilder::new().raw_chunks(chunks).build().await`,
   it will get tracking automatically from the updated `build()`.

7. Add tests in the existing `#[cfg(test)] mod tests`:
   - `test_mock_llm_tracks_received_requests`: Send 2 POSTs with different JSON bodies,
     call `received_requests()`, assert len == 2 and bodies match.
   - `test_mock_llm_response_sequence`: Build with `response_sequence(vec![chunks_a, chunks_b])`,
     send 2 POSTs, assert first response contains text A and second contains text B.
   - `test_mock_llm_response_sequence_repeats_last`: Build with 1-element sequence,
     send 2 POSTs, assert both get the same response.

IMPORTANT: Do NOT change the existing handler's function signature or break existing
test patterns. The handler currently ignores the request body -- now it captures it.
The existing `anthropic_chunks` / `openai_chunks` / `raw_chunks` methods still set
`self.chunks` which is used when `response_sequence` is None.
