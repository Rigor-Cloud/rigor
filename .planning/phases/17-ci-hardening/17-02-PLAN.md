IMPORTANT: Do NOT modify any existing steps. Only ADD new steps and update the permissions block.

**Step 1: Update permissions.**
Change `permissions:` from just `contents: read` to:
```yaml
permissions:
  contents: read
  id-token: write    # Required for cosign keyless OIDC signing
```

The `id-token: write` permission is required for GitHub Actions to issue an OIDC token that cosign uses for keyless signing. This is the ONLY change to existing content -- it is additive (adding a permission line) and does not alter the `contents: read` line.

**Step 2: Add signing steps.**
Insert two new steps AFTER the "Compute SHA256 for each tarball" step and BEFORE the "Extract release notes from CHANGELOG" step:

```yaml
      - name: Install cosign
        uses: sigstore/cosign-installer@v3

      - name: Sign release tarballs (keyless OIDC)
        run: |
          set -euo pipefail
          VERSION="${{ steps.version.outputs.VERSION }}"
          for tarball in dist/rigor-${VERSION}-*.tar.gz; do
            echo "Signing ${tarball}..."
            cosign sign-blob --yes \
              --oidc-issuer https://token.actions.githubusercontent.com \
              --output-signature "${tarball}.sig" \
              --output-certificate "${tarball}.pem" \
              "${tarball}"
          done
          ls -la dist/*.sig dist/*.pem
```

Cosign keyless signing uses the GitHub Actions OIDC token (no private keys to manage). The `--yes` flag skips interactive confirmation. Each tarball gets a `.sig` (detached signature) and `.pem` (signing certificate) file.

**Step 3: Update the `gh release create` command.**
In the existing "Create GitHub Release" step, add the `.sig` and `.pem` files to the upload list. Change the `gh release create` command to include the signature files:

Find the line:
```
            "dist/SHA256SUMS.txt"
```

And add after the SHA256SUMS.txt line (before the closing of the command):
```
            dist/rigor-${VERSION}-*.sig \
            dist/rigor-${VERSION}-*.pem
```

So the full gh release create asset list becomes:
```
            "dist/rigor-${VERSION}-aarch64-apple-darwin.tar.gz" \
            "dist/rigor-${VERSION}-x86_64-apple-darwin.tar.gz" \
            "dist/SHA256SUMS.txt" \
            dist/rigor-${VERSION}-*.sig \
            dist/rigor-${VERSION}-*.pem
```

This is a minimal modification to the existing step -- only adding additional files to the upload list. The step structure, environment, and logic remain identical.
