Create `.github/corpus/injection/run-regression.sh` as an executable bash script that:

1. **Hardcodes the same 9 PATTERNS array** from `.github/workflows/pr-injection-scan.yml` lines 55-63.
   Copy them verbatim -- do not abstract or re-derive. The script is intentionally coupled to the
   workflow so that pattern drift between the two is itself a regression signal.

   ```bash
   PATTERNS=(
     'ignore (all |the )?(previous|prior|above) (instructions|directives|prompts)'
     'disregard (all |the )?(previous|prior|above)'
     '(you are now|you will now) (acting as|pretending to be|a jailbroken)'
     'system[: ]+you (must|will|should) (ignore|forget|override)'
     'developer (override|mode)[: ]'
     '</?(system|instructions|prompt)>'
     '<\|im_(start|end)\|>'
     '\[\[(system|override|admin)\]\]'
     'BEGIN (SYSTEM|PROMPT|INSTRUCTIONS?) OVERRIDE'
   )
   ```

2. **Resolves SCRIPT_DIR** using `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"` so
   the script works when invoked from any directory.

3. **Positive scan loop:**
   - Iterates every `.txt` file in `$SCRIPT_DIR/positives/`.
   - For each file, reads every non-comment line (skip lines starting with `#`).
   - Tests each line against every pattern using `grep -qiE "$pat"`.
   - If at least one line in the file matches at least one pattern, the file PASSES.
   - If NO line matches ANY pattern, the file FAILS and is recorded.
   - Print per-file result: `PASS: positives/pat01-...-01.txt (matched pattern N)` or
     `FAIL: positives/pat01-...-01.txt (no pattern matched)`.

4. **Negative scan loop:**
   - Iterates every `.txt` file in `$SCRIPT_DIR/negatives/`.
   - For each file, reads every non-comment line.
   - Tests each line against every pattern using `grep -qiE "$pat"`.
   - If ANY line matches ANY pattern, the file FAILS (false positive) and is recorded.
   - If no line matches, the file PASSES.
   - Print per-file result: `PASS: negatives/benign-...-01.txt (clean)` or
     `FAIL: negatives/benign-...-01.txt (false positive on pattern N)`.

5. **30KB cap test:**
   - Reads `$SCRIPT_DIR/negatives/benign-30kb-padding.txt`.
   - Asserts `wc -c < file` is greater than 30000.
   - Truncates content to 30000 bytes with `head -c 30000` and runs it through patterns to
     confirm it stays clean even when truncated. This mirrors the workflow's `head -c 30000` cap.
   - Print: `PASS: 30KB cap test (file is {N} bytes, truncated to 30000, clean)`.

6. **Summary and exit code:**
   - Print total counts: `{passed}/{total} positives passed, {passed}/{total} negatives passed`.
   - If any positive FAILED or any negative FAILED: exit 1.
   - If all pass: exit 0.

7. **Script header:**
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   #
   # Regression test for pr-injection-scan.yml regex patterns.
   # Runs the same 9 patterns against committed fixture corpus.
   # Exit 0 = all fixtures behave as expected. Exit 1 = regression detected.
   #
   # Usage: bash .github/corpus/injection/run-regression.sh
   #        (or from any directory: bash /path/to/run-regression.sh)
   #
   ```

8. **Make executable:** `chmod +x .github/corpus/injection/run-regression.sh`.

After creating the script, run it and confirm exit code 0. If any fixture fails, fix the
fixture content (not the patterns) to ensure correct coverage. The patterns are sacrosanct.
