    let config = load_rigor_config(config_path).expect("load rigor.yaml");

Create `crates/rigor/src/cli/corpus.rs` following the `cli/refine.rs` pattern exactly.

**CorpusCommands enum** (derive Subcommand):
- `Record` variant with args: `--prompts` (PathBuf, default `.planning/corpus/prompts`), `--models` (String, required), `--samples` (u32, default 10), `--temperature` (f64, default 0.7), `--max-tokens` (u32, default 512), `--output` (PathBuf, default `.planning/corpus/recordings`), `--resume` (bool flag), `--prompt` (Option String, filter to single prompt ID)
- `Stats` variant with args: `--recordings` (PathBuf, default `.planning/corpus/recordings`), `--rigor-yaml` (Option PathBuf, optional path to rigor.yaml)
- `Validate` variant with args: `--prompts` (PathBuf, default `.planning/corpus/prompts`), `--recordings` (PathBuf, default `.planning/corpus/recordings`)

**run_corpus_command dispatcher:** Match on CorpusCommands, call the appropriate handler.

**run_record handler (REQ-010):**
- Parse `--models` as comma-separated, bail if empty
- Call `crate::corpus::load_prompts` to load manifests
- Call `crate::corpus::OpenRouterClient::from_env()` for the HTTP client
- Build `RecordConfig` from CLI args
- Create tokio runtime inline (`tokio::runtime::Runtime::new().unwrap()`) matching `cli/ground.rs:467`
- In the async block, iterate manifests, skip if `--prompt` filter doesn't match, call `crate::corpus::record_prompt`, print per-manifest stats to stderr

**run_stats handler (REQ-011):**
- Call `crate::corpus::load_recordings` on the `--recordings` path
- Build the replay function: if `--rigor-yaml` is provided use that, otherwise try `super::find_rigor_yaml(None)`. If rigor.yaml is found, use the replay pattern from `tests/corpus_replay.rs`: `load_rigor_config`, `PolicyEngine::new`, `extract_claims_from_text`, check `violated`. If rigor.yaml is NOT found, warn to stderr and use pass-through `|_| false`.
- Call `crate::corpus::compute_stats(recordings, replay_fn)` and `crate::corpus::aggregate_by_model`
- Serialize as JSON to stdout with structure `{ "per_prompt": [...], "per_model": [...] }`
- Use serde_json::json! macro or define a small output struct with Serialize

**run_validate handler (REQ-012):**
- Call `crate::corpus::load_prompts` and `crate::corpus::load_recordings`
- Build a HashMap from manifests by ID for O(1) lookup
- Iterate all recordings. For each sample: use `sample.model` (the original unslugged model name) and `sample.temperature` to call `crate::corpus::record::compute_prompt_hash(manifest, &sample.model, sample.temperature)`. Compare against `sample.prompt_hash`. Also check `sample.response_text` is non-empty. Collect all errors.
- Print OK count + error count. Exit with anyhow::bail if errors found.

**Use `anyhow::Result` throughout.** No custom error types.
