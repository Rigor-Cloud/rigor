**Part A -- Evaluator fail-open tests (gap 5):**

Add these tests to the EXISTING `mod tests` block in `evaluator/pipeline.rs`:

1. `test_failing_evaluator_returns_allow` -- Create a `struct FailingEvaluator;` that implements `ClaimEvaluator` where `can_evaluate` always returns true and `evaluate` returns `EvalResult::allow("internal error (fail-open): simulated failure")`. Register it in a pipeline, evaluate a claim, assert `!result.violated` and `result.reason` contains "fail-open". This verifies the fail-open contract documented in the trait docstring (line 74-75: "On internal error, prefer returning an `allow` result").

2. `test_regex_evaluator_fail_open_on_engine_error` -- This tests the actual RegexEvaluator error path. The RegexEvaluator::evaluate catches `Err(e)` from `engine.evaluate()` and returns `EvalResult::allow(...)`. To trigger this: construct a RegexEvaluator normally, then evaluate a claim against a constraint that has invalid Rego (e.g., rego field set to `"invalid { rego syntax"` -- note: PolicyEngine::new skips invalid constraints but returns Ok). Since PolicyEngine::new skips bad constraints gracefully, the engine.evaluate call may return Ok([]) instead of Err. If that's the case, verify the alternative path: `EvalResult::allow(format!("no violation of {}", constraint.id))` is returned. The key assertion is: with no matching evaluator or an error, the result is NEVER `violated: true`.

3. `test_all_evaluators_miss_returns_allow` -- Pipeline with no evaluators registered, no fallback. Evaluate a claim against a constraint. Assert result is not violated and reason contains "no evaluator matched". (Complements existing `empty_pipeline_without_fallback_is_permissive` but names it explicitly as a fail-open test.)

The `FailingEvaluator` struct should be defined inside the test module (not in production code).
