# EC-12: Migrate `~/.rigor/memory.json` (MemoryStore) into belief_events; deprecate JSON file

> Part of umbrella: #34 [UMBRELLA] Epistemic Cortex
> Depends on: **EC-10**
> Lands in: `crates/rigor/src/memory/episodic.rs` migration tool + `crates/rigor/src/cli/epistemic.rs`

## Scope

The cleanup step. After this lands:

- A one-shot migrator reads `~/.rigor/memory.json` (if present) and emits backfill events into `belief_events`.
- Episodic entries (`SessionEpisode`) convert to `SessionStarted` + summary beliefs; semantic entries (`PathPattern`, `ModelPattern`) convert to testimonial-kind beliefs.
- `rigor epistemic migrate --from memory.json` is the CLI entry. Idempotent (uses deterministic event IDs so re-runs are no-ops).
- After successful migration, the JSON file is renamed to `~/.rigor/memory.json.migrated-TIMESTAMP` (not deleted â€” safety net for rollback).
- `MemoryStore` struct and `episodic.rs` are deprecated but kept compilable. A follow-up release removes them.
- `rigor refine` queries unify: both legacy memory data and new belief data flow from `belief_events`.

## Design constraints pinned from the design thread

- **One-shot migration.** Not a runtime shim; not a dual-write. Migrate â†’ deprecate â†’ eventually remove.
- **Idempotent.** Running twice must produce the same belief_events rows. Achieved via canonical event_id over legacy content.
- **Preserve JSON file as backup.** Rename, don't delete. A rollback is always possible.
- **Retain compile-compat for one release.** `MemoryStore::load()` still works until the next minor. Gives callers time to migrate.
- **No data loss.** Every episode, every path pattern, every model pattern becomes at least one event.

## Mapping (authoritative)

### `SessionEpisode` â†’ events

Given a legacy episode:

```json
{
  "session_id": "abc123",
  "timestamp": "2026-03-15T...",
  "git_commit": "def456",
  "total_violations": 5,
  "constraint_ids": ["rust-no-gc", "tokio-is-async"],
  "sample_claims": ["Rust uses GC", "tokio uses green threads"],
  "outcome": "verified"
}
```

Emit:
1. `SessionStarted { detection_method=PrefixHash, agent_kind=Unknown }` with `session_id=abc123`, `timestamp=<parsed>`, `git_commit=def456`.
2. For each `sample_claim`: `BeliefAsserted { kind: Claim, knowledge_type: Memory, payload_json: { text: <claim>, source: 'legacy-migration' }, base_strength: 0.5, source_id: 'legacy-migration' }`.
3. If `outcome == 'verified'`: `BeliefVerified { method: Human, at_commit: def456 }` for each claim.
4. If `outcome == 'false_positive_dominant'`: `Inhibited { reason: Manual, cause: legacy migration }`.
5. `SessionEnded` with derived timestamp.

### `PathPattern` â†’ events

```json
{
  "path": "src/daemon/proxy.rs",
  "constraint_hits": { "rust-no-gc": 3, "tokio-is-async": 1 },
  "total_hits": 4
}
```

Emit (one per (path, constraint_id) pair):
- `BeliefAsserted { kind: Justification, knowledge_type: Memory, payload_json: { anchor_path: <path>, constraint_id: <cid>, hit_count: N }, base_strength: 0.6, source_id: 'rigor-map-legacy' }`.

### `ModelPattern` â†’ events

```json
{
  "model": "claude-sonnet-4",
  "constraint_hits": { "rust-no-gc": 17 },
  "false_positives": 2,
  "true_positives": 15
}
```

Emit:
- For each (model, constraint_id, count): series of `BeliefAsserted` events attributed to the model's `source_id`.
- Update `sources.total_contributions`, `accurate_count`, `contradicted_count` via direct SQL (using the ledger's outcome functions in bulk).

## Implementation notes & invariants

**Invariant 1: canonical event_id makes migration idempotent.** If `MemoryJsonMigrator::migrate` is run twice over the same JSON, the second run returns all same event_ids â†’ inserts are no-ops â†’ `duplicates_skipped` counts what was not re-appended.

**Invariant 2: legacy timestamps preserved.** Migration does NOT use `now()` â€” it uses the timestamps from the JSON. This preserves the temporal ordering of the audit trail.

**Invariant 3: rename is atomic.** `fs::rename(~/.rigor/memory.json, ~/.rigor/memory.json.migrated-<ts>)` is atomic on POSIX filesystems. Success of migration implies the rename.

**Invariant 4: dry-run writes nothing.** Not even a transaction open; computes the report in memory.

**Invariant 5: legacy-source auto-registered.** If `sources` table doesn't have `legacy-migration` and `rigor-map-legacy`, migration inserts them with credibility 0.6 and 0.8 respectively (lower than default model credibility â€” legacy data is testimonial, not freshly-verified).

**Invariant 6: partial failures recoverable.** If migration fails mid-way, partial events already appended persist. Re-running picks up where it left off (idempotent) â€” no manual cleanup required.

**Operational detail: marker file.** After successful migration, write `~/.rigor/.migrated-from-memory-json` with the source JSON's SHA-256. Subsequent runs without `--force` refuse to re-migrate the same content.

**Operational detail: deprecated `MemoryStore`.** Add `#[deprecated(note="Use EpistemicCortex instead; run `rigor epistemic migrate`")]` on public items. Doesn't break compiles; triggers warnings.

## Additional items surfaced in review

- **Malformed JSON handling.** Test: `test_malformed_json_rejected_with_clear_error` â€” source file is truncated mid-object. Migrator returns Err naming the file, the line/column, and the parse error. Does NOT partial-migrate. Does NOT rename the source file. Errors become a visible item in `MigrationReport.errors` AND cause the CLI to exit non-zero.
- **Top-level `last_updated` field preserved as metadata event.** `MemoryStore` has a `last_updated: Option<String>` timestamp. Migration emits a synthetic event at the very end: `BeliefAsserted { kind: Memory, payload: { "type": "migration_marker", "legacy_last_updated": "...", "source_file_sha256": "..." }, source_id: "legacy-migration" }`. Audit trail preserves the metadata without losing it.
- **Partial migration state is safe.** Test: `test_partial_migration_leaves_consistent_state_on_error` â€” simulate DB outage after migrating 50% of episodes. Re-run migration. Idempotent event IDs mean the second run picks up where the first left off; no duplicates, no gaps. Final state matches a single complete run.
- **Dry-run produces structurally identical report.** Test: `test_dry_run_report_matches_real_run_counts` â€” dry-run vs. real run against same input JSON; all count fields match.
- **Marker-file SHA-256 includes whitespace.** The marker records the exact byte-for-byte SHA of the source JSON. Whitespace changes â†’ SHA changes â†’ `--force` is required, preventing accidental re-migration after manual tweaks.
- **CLI output verbosity.** `rigor epistemic migrate` produces machine-readable (JSON) output when invoked via `--json`, otherwise human-readable text. Test `test_cli_json_output_schema`.
- **Observability (X-1).** `cortex.migrate.memory_json` span with attributes matching `MigrationReport` fields. Per-record span `cortex.migrate.record` when debug-logging is enabled (off by default, too noisy for production).
- **Backward compatibility test.** Load a known-good historical `memory.json` fixture (`tests/fixtures/historical_memory_v2_20.json` or similar), migrate, verify exact expected events. Regression guard against accidental schema drift in `MemoryStore` reader.
- **No-recursion note.** One-shot CLI; no LLM calls; no X-2 concern.
